# 主体管理改造项目计划

更新时间：2026-07-08

## 1. 当前结论

本项目已完成现有主体管理列表页盘点、线上页面观察、Figma 既有页面绘制、配置页第一版设计草稿，以及 `pic/` 参考图复盘。

最新方向调整为：以 Figma「评审修正版」6 个 frame 作为当前权威设计稿，继续迭代“主体详情态 - 账号中心配置页”。旧 frame `900:5340` 仅作为历史草稿参考；下一版不再沿用“一页完整总览配置”的表达方式，而是改为“主体详情框架 + 账号中心 Tab + 左侧二级菜单 + 右侧单模块配置页”的后台配置形态。

项目当前阶段定位为产品经理本地文档评审与 Figma 设计稿修订。

## 3. 最新页面方向

| 区域 | 决策 |
| --- | --- |
| 页面外壳 | 复用现有平台服务中心顶部导航、左侧系统菜单、主体管理选中态 |
| 主体详情顶部 | 所有 6 个模块页一致展示左上无边框 chevron 回退、LOGO、主体名称、主体 ID、账号体系 ID、最近更新人、最近更新时间、保存 |
| 配置类型 | 使用 Tabs，默认包含“账号中心”，支持新增配置类型 |
| 账号中心内部 | 使用左侧二级菜单切换配置模块，右侧只展示当前模块内容 |
| 二级菜单 | 品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 页面形态 | 单模块切换式后台，不在一个页面铺开所有配置 |
| 账号注销 | 先保留菜单与占位页，不编造未确认字段 |

## 6. 最新验收点

| 场景 | 验收标准 |
| --- | --- |
| 点击查看 | 当前页进入主体详情态，不新开页面 |
| 返回主体列表 | 可通过左上无边框 chevron 矢量回退图标回到主体列表，不使用右上“返回列表”按钮，不使用文本箭头 |
| 配置类型 | 默认展示账号中心 Tab，并保留新增配置类型入口 |
| 二级菜单 | 左侧展示品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 通行证能力 | 位于二级菜单第二位，包含账密登录、绑定手机、绑定邮箱三块配置卡片 |
| 单模块展示 | 右侧只展示当前选中模块，不铺开全部配置 |
| 品牌与合规 | 对齐参考图，包含 LOGO、备案版权、协议入口 |
| 游戏配置 | 对齐参考图，使用表格管理游戏；账号申诉、账号找回、账号冻结用开启/关闭状态卡片表达 |
| 账号申诉 | 对齐参考图，重点展示申诉条款和申诉须知，不重复编辑游戏配置 |
| 问题帮助 | 对齐参考图，FAQ 置顶，FAQ 表格下方提供 `+ 添加FAQ` 入口，客服入口放在 FAQ 之后 |
| 账号注销 | 保留占位，不编造字段 |
| 视觉质量 | 无异常换行、错位、遮挡、跨列、内容溢出 |
