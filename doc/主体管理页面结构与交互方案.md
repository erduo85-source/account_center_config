# 主体管理页面结构与交互方案

更新时间：2026-07-08

## 1. 当前结论

主体管理新增功能基于现有列表页继续扩展。点击表格操作列“查看”后，在当前页进入主体详情态，不新开页面。

详情态最新结构为：

1. 顶部主体信息区
2. 配置类型 Tabs
3. 账号中心 Tab 内的左侧二级菜单
4. 右侧当前模块内容

下一版不再采用“一页完整总览配置”或左右双栏瀑布式布局。后台使用者每次只处理一个配置模块，降低扫描和误配成本。

本轮只沉淀页面方案并指导 Figma 设计稿调整，不展开技术实现。

## 3. 列表态保留规则

| 区域 | 保留规则 |
| --- | --- |
| 顶部导航 | 保留平台服务中心全局导航，`项目管理` 为当前一级导航 |
| 全局地区 | 顶栏右上角地区下拉作为国内/海外唯一切换入口 |
| 左侧系统菜单 | 保留系统管理、服务端配置菜单结构，`主体管理` 为选中项 |
| 页面标题 | 保留 `主体管理` 与说明文案 |
| 筛选区 | 保留主体名称搜索和地区展示；地区控件同步全局地区，不作为第二套独立切换 |
| 表格 | 保留主体列表列结构、密度、分页和操作列 |
| 查看入口 | 点击 `查看` 进入当前页详情态 |

## 4. 详情态结构

| 层级 | 区域 | 内容 |
| --- | --- | --- |
| 1 | 主体信息区 | 所有 6 个模块页一致展示：左上无边框 chevron 回退、LOGO、主体名称、主体 ID、账号体系 ID、最近更新人、最近更新时间、保存 |
| 2 | 配置类型 Tabs | 默认 `账号中心`，可新增配置类型 |
| 3 | 账号中心二级菜单 | 品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 4 | 右侧内容区 | 当前选中菜单对应的单模块配置页 |

## 5. 二级菜单定义

| 菜单项 | 页面职责 | 当前处理 |
| --- | --- | --- |
| 品牌与合规 | 配置账号中心前台品牌露出、备案版权、协议入口 | 对齐 `pic/品牌与合规.jpg` |
| 通行证能力 | 配置账密登录、绑定手机、绑定邮箱等通行证能力 | 新增为二级菜单第二项 |
| 游戏配置 | 配置账号申诉、账号找回、账号冻结等能力关联的游戏列表，能力项用开启/关闭状态卡片表达 | 对齐 `pic/游戏配置.jpg` |

| 账号注销 | 配置账号注销相关内容 | 暂无真实参考，保留占位页 |
| 问题帮助 | 配置 FAQ 和客服入口，FAQ 置顶并支持继续添加 | 参考 `pic/账号注销.jpg`，该图实际展示问题帮助页面；FAQ 表格下方保留 `+ 添加FAQ` |

## 6. 关键交互规则

| 交互 | 规则 |
| --- | --- |
| 进入详情 | 点击列表 `查看` 后，当前内容区切换为详情态 |
| 返回主体列表 | 点击左上无边框 chevron 矢量图标返回主体列表；不使用右上“返回列表”，不使用文本箭头 |
| 切换配置类型 | Tabs 切换配置类型；账号中心为默认配置类型 |
| 新增配置类型 | 点击 `新增配置类型` 创建自定义配置类型，后续字段另行定义 |
| 切换二级菜单 | 左侧菜单切换右侧单模块内容，不在同一页铺开全部配置 |
| 地区切换 | 使用全局地区下拉；地区切换影响列表和详情数据隔离 |

## 7. 明确不采用的方案

| 旧方案 | 不采用原因 |
| --- | --- |
| 一页铺开全部配置 | 信息量过高，不像真实后台操作页 |
| 左右双栏总览 | 扫描路径不稳定，模块权重混乱 |
| 安全中心重复展示游戏配置表 | 游戏配置应集中维护，账号申诉/账号找回只引用其结果 |
| 账号注销编造字段 | 当前缺少真实参考，先占位 |
| 顶部警示条常驻 | 当前设计已删除顶部警示条，避免干扰 |

## 9. 验收点

| 场景 | 验收标准 |
| --- | --- |
| 列表态 | 与现有 Figma 列表页视觉基准一致 |
| 详情态 | 进入详情后保持后台框架，不跳出系统 |
| 返回主体列表 | 使用左上 chevron 矢量回退图标，不在右上放“返回列表”按钮，不使用文本箭头 |
| 二级菜单 | 菜单项固定且选中态清晰 |
| 通行证能力 | 位于二级菜单第二位，包含账密登录、绑定手机、绑定邮箱三块配置卡片 |
| 单模块内容 | 右侧只展示当前模块，不出现全部模块总览 |
| 参考图对齐 | 品牌与合规、游戏配置、账号申诉、问题帮助与 `pic/` 参考一致 |
| 账号注销 | 只展示占位，不出现未确认字段 |
| 视觉质量 | 无错位、无遮挡、无跨列、表格列宽稳定 |
