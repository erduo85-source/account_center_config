# 主体管理账号中心配置页评审意见需求对齐稿

更新时间：2026-07-08

## 1. 结论

本轮只把内部评审意见转成可执行的页面方案，用于更新需求文档并指导未完成的 Figma 设计稿调整。

当前阶段不制作可交互页面，不定义真实保存规则，不展开落地细节。

## 2. 评审意见处理

| 评审意见 | 处理结论 | 落点 | 状态 |
| --- | --- | --- | --- |
| 配置页不使用右上“返回列表”按钮，改为左上回退按钮 | 采纳；使用当前 Figma 最新的浏览器回退式 chevron 矢量图标 | 主体详情页左上角 | 已确定 |
| 游戏配置页中账号申诉、账号找回、账号冻结等项，图标改为状态卡片，包括开启和关闭 | 采纳 | 游戏配置列表能力列 | 已确定 |
| FAQ 的重要性大于客服跳转，因此 FAQ 置顶 | 采纳 | 问题帮助页 | 已确定 |
| 账密登录、登录手机、安全手机、安全邮箱不放顶部，需要放到账号中心下配置 | 采纳，新增“通行证能力”模块承载 | 账号中心二级菜单第二位 | 已确定 |

## 3. 页面结构

| 区域 | 页面方案 |
| --- | --- |
| 顶部主体信息区 | 所有 6 个配置页一致展示：左上无边框 chevron 回退、LOGO、主体名称、主体 ID、账号体系 ID、最近更新人、最近更新时间、保存入口 |
| 配置类型 Tabs | 默认展示账号中心，保留新增配置类型入口 |
| 账号中心二级菜单 | 品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 右侧内容区 | 根据左侧菜单展示当前单模块内容 |

顶部主体信息区不展示账密登录、登录手机、安全手机、安全邮箱等能力标签。

## 4. 模块方案

| 模块 | 页面内容 | Figma 调整要点 |
| --- | --- | --- |
| 品牌与合规 | 品牌 LOGO、备案版权、协议入口 | 延续现有参考图结构 |
| 通行证能力 | 账密登录、绑定手机、绑定邮箱三块配置卡片 | 新增为二级菜单第二项；使用配置卡片承载能力项 |
| 游戏配置 | 游戏列表、项目/appid、账号申诉、账号找回、账号冻结、更新时间/人、操作 | 三个能力列使用开启/关闭状态卡片，不再只用图标 |
| 账号申诉 | 申诉条款、申诉条款链接、申诉须知富文本 | 不重复展示游戏配置表 |
| 账号注销 | 待补充占位 | 不编造字段 |
| 问题帮助 | FAQ 表格、`+ 添加FAQ` 虚线整行按钮、客服展示文案、客服跳转链接 | FAQ 置顶，FAQ 支持继续添加，客服跳转配置放在 FAQ 之后 |

## 5. 通行证能力细化

| 配置卡片 | 配置项 | 说明 |
| --- | --- | --- |
| 账密登录 | 账密登录 | 单独表达账号密码登录能力 |
| 绑定手机 | 登录手机、安全手机 | 两个配置项放在同一张绑定手机卡片内 |
| 绑定邮箱 | 安全邮箱 | 首版只包含安全邮箱，不预留未确认的登录邮箱 |

## 6. Figma 调整方向

| 调整项 | 设计稿处理 |
| --- | --- |
| 左上回退 | 移除右上“返回列表”按钮，在主体详情区域左上放浏览器回退式 chevron 矢量图标，不使用文本箭头 |
| 通行证能力 | 在左侧二级菜单新增“通行证能力”，位置在“品牌与合规”之后 |
| 顶部能力标签 | 从顶部主体信息区移除账密登录、登录手机、安全手机、安全邮箱标签 |
| 游戏能力状态 | 账号申诉、账号找回、账号冻结使用开启/关闭状态卡片 |
| 问题帮助排序 | FAQ 表格放在页面顶部，表格下方提供 `+ 添加FAQ` 入口，客服跳转配置放在其后 |

状态卡片在 Figma 中只需表达开启和关闭两种状态；是否点击切换不在本轮定义。
