## 2. 整体布局规范

| 区域 | 规范 |
| --- | --- |
| 顶部导航 | 复用平台服务中心深色顶部导航，`项目管理` 为当前一级导航 |
| 左侧系统菜单 | 复用系统管理/服务端配置侧边栏，`主体管理` 为选中项 |
| 内容背景 | 使用现有浅灰后台背景，内容卡片使用白底 |
| 列表态 | 保持 `845:1317` 的表格密度、搜索区、分页和操作列 |
| 详情态 | 当前页内容区替换为主体详情，不新开页面 |
| 配置区 | 账号中心 Tab 内使用左侧二级菜单 + 右侧单模块内容 |

## 4. 字体与密度

| 场景 | 字号 | 规则 |
| --- | ---: | --- |
| 顶部导航 | 16 | 延续现有导航 |
| 页面主标题 | 18 | 蓝色标题用于列表页标题 |
| 区块标题 | 16 | 配置模块标题，左侧带蓝色装饰线 |
| 表格表头 | 12 | 加粗，单行展示 |
| 表格正文 | 12 | 单行优先，超长省略 |
| 表单标签 | 12-14 | 与参考图保持紧凑 |
| 辅助说明 | 12 | 灰色，不抢主信息 |

## 5. 详情页结构规范

| 区域 | 规范 |
| --- | --- |
| 主体信息区 | 横向布局，左上提供无边框 chevron 矢量回退图标，左侧 LOGO，右侧主体名称、账号体系信息、最近更新人、最近更新时间，保存按钮右对齐 |
| 能力配置位置 | 顶部不展示账密登录、登录手机、安全手机、安全邮箱等标签；这些能力下沉到通行证能力模块 |
| Tabs | 位于主体信息区下方，账号中心为默认 Tab |
| 二级菜单 | 位于配置内容左侧，宽度固定，选中项使用浅蓝底 |
| 右侧内容 | 单栏展示当前模块，宽度撑满剩余区域 |

## 6. 二级菜单规范

| 菜单项 | 选中页内容 |
| --- | --- |
| 品牌与合规 | 品牌 LOGO、备案与版权、协议入口 |
| 通行证能力 | 账密登录、绑定手机、绑定邮箱三块配置卡片 |
| 游戏配置 | 游戏列表表格、添加游戏 |
| 账号申诉 | 申诉条款、申诉须知富文本 |
| 账号注销 | 待补充占位 |
| 问题帮助 | FAQ 表格、`+ 添加FAQ`、客服入口 |

二级菜单是页面导航，不是锚点目录。右侧只展示当前选中菜单对应的模块。

## 7. 模块设计规范

| 模块 | 规范 |
| --- | --- |
| 品牌与合规 | 分为品牌 LOGO、备案与版权、协议入口三块；LOGO 上传位左右排布 |
| 通行证能力 | 使用三块等宽配置卡片：账密登录、绑定手机、绑定邮箱；账密登录展示 `必选` 标签且默认开启禁用，绑定手机内放登录手机和安全手机，绑定邮箱内只放安全邮箱 |
| 游戏配置 | 使用主表格，不把列表整体卡片化；能力列用开启/关闭状态卡片表达；底部提供添加游戏入口 |
| 账号申诉 | 表单区展示条款文案和跳转链接；须知使用富文本编辑器样式 |
| 账号注销 | 使用空态/占位说明，避免编造配置字段 |
| 问题帮助 | FAQ 置顶，FAQ 表格下方放虚线整行 `+ 添加FAQ` 按钮，下面放客服跳转配置；FAQ 支持排序、编辑、删除 |

## 8. 表格规范

| 项 | 规范 |
| --- | --- |
| 表头 | 浅灰背景，12px 加粗 |
| 行高 | 保持后台紧凑密度，避免多行撑高 |
| 操作列 | 使用蓝色编辑、红色删除 |
| 长文本 | 单行省略或在详情/编辑态展示完整内容 |
| 添加入口 | 表格底部可使用虚线整行按钮 |
| 图标/头像 | 游戏图标固定尺寸，不影响行高 |

## 9. 表单与控件规范

| 控件 | 规范 |
| --- | --- |
| Input | 高度与现有后台一致，标签左侧对齐 |
| Select | 用于账号体系、项目多选、游戏选择等 |
| Upload | LOGO 使用 Upload-Picture 样式 |
| Switch | 用于通行证能力等明确的能力开关 |
| Button | 主操作使用蓝色 primary，次操作使用描边按钮 |
| RichText | 账号申诉须知使用工具栏 + 内容区样式 |

## 10. 通行证能力细化规范

| 区域 | 规范 |
| --- | --- |
| 标题区 | 蓝色竖向强调条 + `通行证能力` + info icon + 辅助说明；标题 16px / 600 / `#1F2937` |
| 辅助说明 | 使用 `从顶部下沉到账户中心内部，按能力卡片配置`，字号 12px，颜色 `#6B7280` |
| 右侧操作 | 最新版设计稿不展示 `恢复默认`，标题区右侧保持留白 |
| 卡片布局 | 一行三列等宽，卡片间距 24px，卡片 8px 圆角、24px 内边距、`1px solid #E5E7EB` |
| 卡片 Hover | 边框变为 `#BFD7FF`，阴影轻微增强 |
| 图标 | 40px × 40px，背景 `#EEF4FF`，图标颜色 `#2F7BFF`，圆角 8px |
| 配置区 | 与说明区用 `1px solid #EEF0F3` 分割；配置项高 36px，左文案右开关 |
| 开关 | 开启 `#2F7BFF`，关闭 `#D1D5DB`，禁用背景 `#E5E7EB`，点击区域不小于 40px × 24px |
| 设计说明 | 卡片下方展示浅灰说明块，文案为 `该模块承接顶部移除的能力标签；顶部主体信息区仅保留基础识别信息。` |

| 能力卡片 | 配置项 | 展示规则 |
| --- | --- | --- |
| 账密登录 | 账密登录 | 默认开启；如作为基础能力不可关闭，则使用禁用态开关并展示 `必选` 标签 |
| 绑定手机 | 登录手机、安全手机 | 两个开关允许独立配置 |
| 绑定邮箱 | 安全邮箱 | 当前仅支持配置安全邮箱，不出现登录邮箱配置项 |

## 11. 不采用的视觉方式

| 方式 | 原因 |
| --- | --- |
| 双栏总览 | 模块权重混乱，后台使用者难以快速定位 |
| 全部模块同时铺开 | 信息负担过高，页面边界不清晰 |
| 重复游戏配置表 | 公共配置应集中维护，引用处只展示摘要 |
| 多层卡片嵌套 | 页面显得拥挤且降低扫描效率 |
| 编造账号注销字段 | 当前无真实参考 |
| 顶部堆叠能力标签 | 主体信息区只保留基础识别信息，能力配置下沉到账号中心内部 |

## 12. 视觉验收清单

| 检查项 | 标准 |
| --- | --- |
| 顶部导航 | 高度、背景色、选中态与基准一致 |
| 左侧系统菜单 | 主体管理选中态正确 |
| 主体信息区 | 左上 chevron 回退图标、LOGO、名称、账号体系、最近更新人、最近更新时间、保存按钮不挤压；6 个模块页字段一致 |
| Tabs | 选中态清晰，新增入口不抢主操作 |
| 二级菜单 | 选中项清晰，菜单项不换行 |
| 通行证能力 | 位于二级菜单第二位，三块卡片层级清晰 |
| 右侧内容 | 单模块展示，无跨区、遮挡、重叠 |
| 表格 | 列宽稳定，操作列清晰，文字不异常换行 |
| 表单 | 标签、输入框、说明文案对齐 |
| 底部操作 | 保存按钮可见，说明不遮挡内容 |
| 整体 | 无错位、无遮挡、无内容溢出、无过度卡片化 |
