# 主体管理账号中心配置页最新方案

更新时间：2026-07-08

## 1. 最新决策

主体管理配置页下一版以“单模块切换式后台”为准，不再延续当前完整总览式 frame。
当前 Figma 最新稿以「评审修正版」6 个 frame 为准；旧 `900:5340` 仅作为历史设计草稿参考。

页面结构固定为：

1. 主体详情框架
2. 账号中心 Tab
3. 左侧二级菜单
4. 右侧单模块配置页

## 4. 页面结构

| 区域 | 内容 |
| --- | --- |
| 顶部主体信息 | 所有 6 个模块页一致展示：左上无边框 chevron 回退、LOGO、主体名称、主体 ID、账号体系 ID、最近更新人、最近更新时间、保存 |
| 配置类型 Tabs | 账号中心、其他示例、新增配置类型 |
| 左侧二级菜单 | 品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 右侧模块内容 | 根据二级菜单展示当前模块 |

## 5. 模块内容

| 模块 | 内容 | 交互 |
| --- | --- | --- |
| 品牌与合规 | 顶部 LOGO、底部 LOGO、经营许可、公安备案、版权信息、用户协议、隐私协议 | 上传 LOGO、编辑文案和链接、添加/编辑/删除协议 |
| 通行证能力 | 账密登录、绑定手机、绑定邮箱三块配置卡片；账密登录默认开启并展示 `必选` 标签；绑定手机包含登录手机和安全手机，绑定邮箱只包含安全邮箱 | 开关切换只改变当前页面状态，由右上 `保存` 统一提交；不展示 `恢复默认` |
| 游戏配置 | 游戏图标、游戏名、项目/appid、账号申诉状态卡片、账号找回状态卡片、账号冻结状态卡片、最近更新时间、最近更新人、操作 | 添加游戏、编辑游戏、删除游戏；状态卡片表达开启/关闭 |
| 账号申诉 | 申诉条款文案、申诉条款链接、申诉须知富文本 | 编辑表单与富文本，不重复配置游戏 |
| 账号注销 | 待补充占位 | 保留菜单，不展示未确认字段 |
| 问题帮助 | FAQ 表格、`+ 添加FAQ` 虚线整行按钮、客服展示文案、客服跳转链接 | 新增 FAQ、编辑 FAQ、删除 FAQ、排序；FAQ 置顶，客服配置后置 |

## 9. 验收标准

| 场景 | 标准 |
| --- | --- |
| 页面结构 | 符合主体详情框架 + Tabs + 二级菜单 + 单模块内容 |
| 返回主体列表 | 不使用右上“返回列表”按钮；使用当前 Figma 最新的左上无边框 chevron 矢量回退图标，不使用文本箭头 |
| 通行证能力 | 位于二级菜单第二位，包含账密登录、绑定手机、绑定邮箱三块配置卡片 |
| 品牌与合规 | 与参考图结构一致 |
| 游戏配置 | 使用主表格，不再是小型摘要表；能力列使用开启/关闭状态卡片 |
| 账号申诉 | 展示条款和富文本，不重复游戏配置 |
| 问题帮助 | FAQ 置顶，FAQ 表格下方提供 `+ 添加FAQ` 虚线整行按钮，客服入口放在 FAQ 之后 |
| 账号注销 | 只展示占位，不编造字段 |
| 视觉质量 | 无错位、无遮挡、无跨列、表格列宽稳定 |
