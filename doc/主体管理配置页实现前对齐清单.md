## 2. 已确认决策

| 决策点 | 结论 |
| --- | --- |
| 页面形态 | 单模块切换式后台，不再使用完整总览页 |
| 配置入口 | 点击主体列表 `查看` 后进入当前页详情态 |
| 地区切换 | 使用顶栏右上角全局地区下拉 |
| 配置类型 | 默认 `账号中心` Tab，保留新增配置类型入口 |
| 二级菜单 | 品牌与合规、通行证能力、游戏配置、账号申诉、账号注销、问题帮助 |
| 返回方式 | 不使用右上“返回列表”按钮，改为左上无边框 chevron 矢量回退图标 |
| 问题帮助排序 | FAQ 置顶，客服跳转放在 FAQ 之后 |
| 账号注销 | 保留菜单，右侧做待补充占位，不编造字段 |
| 本轮边界 | 只更新需求文档并指导 Figma 设计稿调整，不展开落地细节 |

## 3. 参考图结论

| 参考图 | 结论 | 对页面的影响 |
| --- | --- | --- |
| `pic/品牌与合规.jpg` | 品牌与合规是完整表单页 | 需要包含品牌 LOGO、备案与版权、协议入口 |
| `pic/游戏配置.jpg` | 游戏配置是表格管理页 | 需要大表格、游戏图标、能力列、更新时间、操作、添加游戏 |
| `pic/账号申诉.jpg` | 账号申诉重点是条款和须知 | 不在申诉页重复编辑游戏配置 |
| `pic/账号注销.jpg` | 图片实际展示问题帮助/FAQ | 用作问题帮助参考，不作为账号注销依据 |

## 4. 页面结构

| 区域 | 结果 |
| --- | --- |
| 页面外壳 | 复用已有顶部导航、左侧系统菜单、主体管理选中态 |
| 主体信息区 | 所有 6 个模块页一致展示：左上无边框 chevron 回退、LOGO、主体名称、主体 ID、账号体系 ID、最近更新人、最近更新时间、保存 |
| 配置 Tabs | 账号中心、其他示例、新增配置类型 |

## 5. 模块内容

| 模块 | 首版内容 | 备注 |
| --- | --- | --- |
| 品牌与合规 | 顶部 LOGO、底部 LOGO、经营许可、公安备案、版权信息、协议入口 | 对齐参考图 |
| 通行证能力 | 账密登录、绑定手机、绑定邮箱三块等宽配置卡片；账密登录默认开启并展示 `必选` 标签，绑定手机包含登录手机和安全手机，绑定邮箱只包含安全邮箱 | 新增二级菜单第二项，承接顶部下沉能力标签；标题区不展示 `恢复默认` |
| 游戏配置 | 游戏列表、项目/appid、账号申诉状态卡片、账号找回状态卡片、账号冻结状态卡片、最近更新时间、最近更新人、编辑/删除、添加游戏 | 作为公共复用配置 |
| 账号申诉 | 申诉条款文案与链接、申诉须知富文本 | 不重复游戏配置表 |
| 账号注销 | 待补充占位 | 缺少真实参考，暂不定义字段 |
| 问题帮助 | FAQ 表格、`+ 添加FAQ` 虚线整行按钮、客服展示文案、客服链接 | 参考 `pic/账号注销.jpg`，FAQ 置顶，客服后置 |

## 9. 通行证能力实现补充

| 项 | 要求 |
| --- | --- |
| 区域标题 | `通行证能力` 左侧使用蓝色竖向强调条，标题旁放 info icon，辅助说明为 `从顶部下沉到账户中心内部，按能力卡片配置` |
| 恢复默认 | 最新版设计稿不展示该按钮；本轮原型不提供恢复默认入口 |
| 卡片布局 | 一行三张等宽卡片，卡片间距 24px，开关统一右对齐 |
| 账密登录 | 展示 `必选` 标签；基础能力默认开启，若不允许关闭则使用禁用态开关 |
| 绑定手机 | 配置项为 `登录手机`、`安全手机`，允许独立配置 |
| 绑定邮箱 | 仅展示 `安全邮箱`，不展示登录邮箱 |
| 保存 | 开关切换不立即保存，统一由右上 `保存` 提交并给出 toast 反馈 |
| 本地设计稿 | `prototype/passport-capability/index.html` |

## 11. 设计约束

| 约束 | 说明 |
| --- | --- |
| 当前阶段 | 只做文档需求对齐和 Figma 设计稿调整指引 |
