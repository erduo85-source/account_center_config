## 2. 同步原则

| 类型 | 处理 |
| --- | --- |
| 已接受反馈 | 同步更新需求对齐稿、最新方案、实现前对齐清单、结构交互方案和设计规范，作为下一版 Figma 调整依据 |
| 待确认反馈 | 只记录真实未定内容，不直接定模块字段或具体交互 |
| 后续设计 | 先改 Figma 页面结构和模块层级，再细化控件样式 |
