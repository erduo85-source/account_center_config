## 6. 设计使用规则

| 规则 | 说明 |
| --- | --- |
| 优先复用 | Figma 中可实例化的 Ant Assets 优先直接放入设计稿 |
| 不伪装 | 未找到或无法实例化的组件，只在文档中作为实现建议，不声称已使用 |
| 通行证能力 | 使用三张配置卡片表达账密登录、绑定手机、绑定邮箱 |
| 表格优先 | 游戏配置、FAQ、协议入口优先用表格结构；游戏能力状态卡片只作为表格单元格表达，不把列表整体卡片化 |
| FAQ 优先 | 问题帮助页先展示 FAQ，FAQ 表格下方提供 `+ 添加FAQ` 入口，再展示客服跳转配置 |
| 单模块优先 | 右侧内容区一次只展示一个配置模块 |
| 密度一致 | 控件高度、表格行高、按钮尺寸延续已有后台页面 |
